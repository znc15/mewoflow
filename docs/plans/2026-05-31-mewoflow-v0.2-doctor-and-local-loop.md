# MewoFlow v0.2 Doctor And Local Loop Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a one-command MewoFlow health check for Claude Code, document the Trellis inspiration/differences, and add the smallest local loop improvements needed after v0.1.

**Architecture:** Keep MewoFlow local-first and deterministic. `mewoflow doctor` checks local files, hook wiring, generated Claude skill command, active task state, and recent Claude search evidence recorded by hooks. `mewoflow init` generates the slash-invocable Claude skill, lightweight specs, and journal files without overwriting user edits.

**Tech Stack:** Node.js, TypeScript, Claude Code hooks/skills, Vitest, Markdown project files.
